About Pump Diagnostic System

Welcome to Pump Diagnostics, a cutting-edge web server designed for real-time machine learning monitoring of centrifugal pumps. This project, titled 'Development of Real-Time Machine Learning Monitoring Web Server for Pump Diagnostics,' is the culmination of my final year project. Project Background: This project focuses on developing a defect prediction system using ML algorithms and real-time monitoring of centrifugal pumps. Centrifugal pumps are widely used in various industrial applications, and failures in these pumps can lead to energy loss, performance decline, or even complete system failure, resulting in significant economic losses. Early diagnosis of pump system conditions is crucial to prevent such failures. By leveraging machine learning models and data acquired from vibration and acoustic sensors, the project aims to accurately detect and diagnose pump defects, reducing downtime and enabling proactive maintenance. Problem Statement: In today's data-driven and IoT-enabled world, traditional manual monitoring for pump diagnostics is becoming outdated. There is a need for an online, real-time machine learning monitoring system that enables engineers to quickly identify and diagnose pump problems. However, reports on the implementation of such systems in pump diagnostics applications are scarce. Objectives: The main objectives of this project are as follows: Deploy a machine learning model that utilizes vibration and acoustic signals in the time-frequency domain to identify flow blockages in centrifugal pumps. Develop a web server with a user interface that can be accessed from any device with an internet connection. The web server will connect to the machine learning model to provide real-time diagnostic status updates for the pump. Implement a system using a Raspberry Pi connected to a sensor, specifically the MPU6050. This system will facilitate the transmission of telemetry data from a Python code running on the Raspberry Pi to an Azure IoT Central application. Scope of Project: The project utilizes an experimental rig in the Vibration Lab laboratory to collect data on flow blockage in a centrifugal pump. The collected data, including vibration, acoustic, and rotational speed measurements, will undergo pre-processing and feature extraction. The data will be used to train a machine learning model based on Support Vector Machine (SVM) in MATLAB and Microsoft Azure Machine Learning. A user interface will be developed using Visual Code Studio to enable interaction with the machine learning model and provide predictions on the health condition of the centrifugal pump. Additionally, a sensor called MPU6050 connected to a Raspberry Pi will be utilized to transmit telemetry data to an Azure IoT Central application. By combining advanced machine learning techniques, real-time monitoring, and web server technology, this project aims to revolutionize pump diagnostics and maintenance practices, enabling engineers to proactively identify and address pump issues, minimize downtime, and optimize overall operational efficiency.
